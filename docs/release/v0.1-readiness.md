# v0.1 Rename Release Readiness

Date: 2026-05-25

## Verdict

Ready to tag `v0.1.1` after these rename-readiness changes are merged to
`main` and the GitHub repository has been renamed to `weskor/agent-machine`.

The codebase now has release packaging, an open-source license, a Homebrew tap,
the Homebrew write secret, dependency vulnerability alerts, private
vulnerability reporting, branch protection, and clearer public docs.

## Ready

- README explains what Agent Machine does, install paths, required credentials,
  project files, commands, local state, development, release, and safety notes.
- `SECURITY.md` gives a private vulnerability reporting path and secret-handling
  expectations.
- `CONTRIBUTING.md` points contributors to the repo-specific development loop.
- `LICENSE` grants the project under the MIT License.
- GitHub issue and PR templates route bug reports, release checks, validation,
  and private security reports.
- `.goreleaser.yaml` builds `agent-machine` archives for macOS and Linux on
  `amd64` and `arm64`, with the installed binary named `am`.
- `.github/workflows/release.yml` runs `make ci` before publishing a tagged
  release.
- `weskor/homebrew-tap` exists, is public, and has a `main` default branch.
- `HOMEBREW_TAP_GITHUB_TOKEN` is configured on `weskor/agent-machine`.
- Dependency vulnerability alerts are enabled on `weskor/agent-machine`.
- `weskor/agent-machine` is public.
- Private vulnerability reporting is enabled on `weskor/agent-machine`.
- `main` branch protection requires strict `go-ci`, one approving review, stale
  review dismissal, and conversation resolution; force-pushes and branch
  deletion are disabled.
- `am --version` works without config, Linear, or GitHub credentials.
- A tracked-file scan found no committed real tokens or private keys.
- `make release-check` and `make release-snapshot` provide pinned local
  GoReleaser validation without requiring a globally installed `goreleaser`
  binary.

## Release Procedure

1. Verify the tree is clean except for intentional release changes:

   ```bash
   git status --short
   ```

2. Run local gates:

   ```bash
   mise exec go -- make ci
   git diff --check
   mise exec go -- make release-check
   mise exec go -- make release-snapshot
   ```

3. Confirm the Homebrew tap exists and the release secret is configured:

   ```bash
   gh repo view weskor/homebrew-tap --json nameWithOwner,isPrivate,url,defaultBranchRef
   gh secret list --repo weskor/agent-machine
   ```

4. Verify public repository settings:

   ```bash
   gh repo view weskor/agent-machine --json nameWithOwner,isPrivate,defaultBranchRef,url
   gh api repos/weskor/agent-machine/private-vulnerability-reporting --method GET --include
   gh api repos/weskor/agent-machine/vulnerability-alerts --method GET --include
   gh api repos/weskor/agent-machine/branches/main/protection \
     --jq '{required_status_checks,required_pull_request_reviews,allow_force_pushes,allow_deletions,required_conversation_resolution}'
   ```

5. Tag the rename release:

   ```bash
   git tag v0.1.1
   git push origin v0.1.1
   ```

6. Verify the GitHub release has archives and `checksums.txt`.

7. Verify installs:

   ```bash
   am --version
   brew install --cask weskor/tap/agent-machine
   am --version
   ```

## Stability Notes

The v0.1 scope should be a conservative first release:

- local-first runner only;
- Linear and GitHub integrations configured by the operator;
- no promise of hosted/cloud operation;
- no unreviewed auto-merge recommendation;
- Homebrew cask for macOS plus GitHub release archives for Linux.
